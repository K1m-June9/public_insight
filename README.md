# PublicInsight
  ### 지식그래프로 연결하고 탐색하는 정책 데이터 플랫폼
  ### https://public-insight.co.kr

## Overview
공공 정책 데이터는 양적으로 풍부하지만, 여러 기관에 분산된 파편화 문제와 데이터 간의 맥락 부재로 인해 시민들이 심층적인 내용을 파악하기 어렵습니다.

PublicInsight는 이러한 문제를 해결하기 위해, 흩어져 있는 이기종 데이터(정형, 비정형, 로그)를 '관계' 중심의 지식 그래프로 재구성합니다. 
사용자는 이 지식 네트워크를 시각적으로 탐색하며 숨겨진 연결고리를 발견하고, 단순한 정보 소비를 넘어 스스로 인사이트를 얻는 새로운 데이터 탐색 경험을 할 수 있습니다.

## Key Features
- 지식 그래프 ETL 파이프라인: 이기종 데이터 소스를 융합하여 Neo4j 지식 그래프를 자동으로 구축하는 데이터 파이프라인
- 그래프 기반 ML 예측 엔진: Node2Vec 모델을 활용하여, 그래프의 구조적 정보를 학습하고 노드 간의 숨겨진 유사성을 예측하는 엔진
- 인터랙티브 지식 그래프 시각화 (마인드맵): 구축된 지식 그래프를 사용자가 직접 확장하며 탐색할 수 있는 인터랙티브 인터페이스
- 사용자 활동 기반 준개인화 추천: 사용자의 최근 활동(북마크, 검색 등)을 'Seed'로 삼아 잠재적 관심사를 예측하고 맞춤형 콘텐츠를 추천
- AI 기반 정책 문서 자동 요약: KoBART 모델을 활용하여 긴 원문을 핵심 내용 중심으로 요약, 정보 접근성 향상

## Stack
#### Backend
- FastAPI: 비동기 처리를 통한 고성능 API 서버 구축
- SQLAlchemy: Pythonic한 비동기 ORM을 통한 데이터베이스 상호작용
- APScheduler: ETL 및 ML 파이프라인의 주기적인 실행을 위한 스케줄러

#### Frontend
- React (Next.js): 서버 사이드 렌더링을 활용한 반응형 UI/UX 구축
- TypeScript: 정적 타이핑을 통한 코드 안정성 및 유지보수성 확보
- Tailwind CSS: Utility-First 접근 방식을 통한 빠르고 일관된 UI 개발

#### Database & Storage
- MySQL: 핵심 정형 데이터 저장을 위한 관계형 데이터베이스
- Neo4j: '관계' 중심의 데이터 모델링 및 탐색을 위한 그래프 데이터베이스
- Elasticsearch: 전문(Full-text) 검색 및 사용자 검색 로그 수집/분석
- Redis: 인증 토큰 관리 및 캐싱을 위한 인메모리 데이터 저장소

#### NLP & ML
- KoBART: 한국어 텍스트 요약을 위한 사전 학습된 생성 모델
- Scikit-learn / Kiwi: TF-IDF 기반의 핵심 키워드 추출 및 유사도 계산
- Node2Vec: 지식 그래프의 구조적 정보를 학습하기 위한 그래프 임베딩 모델

#### Infrastructure & Monitoring
- Docker: 애플리케이션 및 모든 서비스의 컨테이너화를 통한 일관된 배포 환경 구축
- Nginx: 리버스 프록시 및 정적 파일 서빙
- Kibana / Grafana: 시스템 로그 및 메트릭 데이터의 실시간 시각화 및 모니터링

## Future Roadmap
지속적인 발전을 위해 시스템을 고도화할 계획입니다.
#### ML 파이프라인 고도화
- 모델 성능 검증 자동화: '챔피언-도전자' 모델 관리 시스템을 도입하여, 재학습된 모델의 성능을 자동으로 평가하고 안정적인 모델 배포 지향
- 하이퍼파라미터 최적화: Node2Vec의 p, q 파라미터 등 핵심 하이퍼파라미터에 대한 체계적인 튜닝을 통해 예측 정확도 향상

#### 지식 그래프 확장 및 심화
- 관계 추출 도입: 문장 내 '주체-행위-대상(SPO)' 관계를 추출하여 그래프의 의미적 표현 능력을 강화
- 시간적 차원 추가: 정책의 발표 시점, 개정 이력 등을 그래프에 반영하여 시계열 분석 기능 구현

#### 서비스 아키텍처 개선
- 데이터 파이프라인 고도화: 현재 단일 스크립트 기반의 ETL 파이프라인을 Airflow 등 워크플로우 관리 도구를 도입하여, 각 단계의 모니터링 및 재실행이 용이한 구조로 개선
- 점진적 업데이트 도입: '전체 재구성' 방식의 확장성 한계를 극복하기 위해, 변경된 데이터만 식별하여 그래프를 부분적으로 업데이트하는 방식으로 전환
- 마이크로서비스 아키텍처 전환 검토: 향후 기능 확장 및 트래픽 증가에 유연하게 대응하기 위해, 모놀리식 아키텍처를 주요 도메인 단위의 마이크로서비스로 분리하는 장기적인 전환을 검토

#### 코드 품질 및 유지보수성 향상
- 코드 리팩토링: 지속적인 코드 리뷰 및 리팩토링을 통해 코드의 가독성, 효율성 및 테스트 커버리지 개선
- 레거시 코드 관리: 기능 변경에 따라 더 이상 사용되지 않는 코드 및 주석을 주기적으로 제거하여 코드베이스의 청결성 유지
